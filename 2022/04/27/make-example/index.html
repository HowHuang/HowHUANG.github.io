Makefile writing example

Revision of class MAE5032-HPC-by_Ju-LIU (https://ju-liu.github.io/)

Version 1

把目标文件bar.o和依赖文件bar.c、bar.h连接起来

make会执行全部命令，make bar.o会仅执行生成bar.o

如果名字不是默认的makefile或者Makefile，用make -f my _script

Version 1.2

如果不喜欢把command打印出来，可以加上@符号

添加一个清理功能，造一个clean的伪目标，伪目标的命令是rm -rf ...

Version 2

用$(wildcard *.c)把本地所有以.c结尾的文件读取成一个列表，赋值给SRC

用$(patsubst %.c,%.o,$(SRC))把SRC中所有以.c结尾的文件名换成以.o结尾，赋值给OBJ

用EXEC表示输出的可执行文件名，用CC表示编译器的名称

因此，下面的也换成变量名

Version 2.1

用makefile自带的一些规则

$@表示规则中的目标文件（the target of the current rule）

$^表示依赖关系中的所有依赖文件（all the prerequisites of the current rule）

$<表示依赖关系中的第一个文件（the first prerequisite of the current rule）

Version 2.2

再把%.o、%.c替换代表所有的.o文件和.c文件

bar.h文件再写成目标文件依赖于.h文件$(OBJ) : *.h

并添加上注释

Version 3

让功能更好一点

用CFLAGS表示编译选项，

用LDFLAGS和LDLIBS表示还需要链接的库（此例为在/usr/lib中m.a或者m.so的库），

并把它们放到# RULES里面

再加一些伪目标clobber、neat、echo

clobber依赖于clean会调用clean并执行rm -rf $(EXEC)，

neat会删除所有以~结尾 或 以 .开头~结尾的文件；这些文件一般是文本编辑器在编辑时产生的一些备份文件，

echo将产生的目标文件在命令行展示出来

Version 4

在ppt 26页，makefile还有一些自带的定义

如$(LINK.o) = $(CC) $(LDFLAGS) $(TARGET_ARCH)、$(COMPILE.c) = $(CC) $(CFLAGS) $(CPPFLAGS)$(TARGET_ARCH) -c可以将RULES进一步简化

（$(TARGET_ARCH)默认是空的，没有定义就没有任何效果，可以用于根据cpu优化，例如xhost等；此例没有用到CPPFLAGS）

注意，=是会展开遍历变量中的所有值，而:=是恒等于、赋值的意思。

用make -p可以查看makefile所有隐含的规则，用make -p | grep "CC"可以去找需要的隐含规则
