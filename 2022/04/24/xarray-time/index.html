I have failed to convert datetime format from ‘float’ (such as 199601) to ‘datetime64’ when input multiple nc. files by using the stupid xr.open_mfdataset .My solution: Here, the DataArray taken as an example is dataset[‘time’].There are three functions I tried to used.

  1. using datetime.strptime(dataset['time'],'%Y-%m') after from datetime import datetime, but I got the Error: strptime() argument 1 must be str, not DataArray. This indicated the dataset[‘time’] is not the valid input.
  2. using pd.to_datetime(dataset['time'], format='%Y%m') after import pandas as pd, but the Error of unhashable type: ‘DataArray’ will follow, which also indicated the dataset[‘time’] is not the valid input.
  3. using dataset['time'].astype('datetime64[ns]') which also from pandas. It works, but the output is not what I want. For example, you will directly get the stupid things like ‘1970-01-01T00:00:00.000199601’ from ‘199601’. However, any ‘datetime64[]’ could not give the correct years and months back to you. So, I tried to use .values
    to return the values from dataset[‘time’]. However, using dataset['time'].astype(int).astype('str').values.astype('datetime64[Y]') can only return the ‘199601’, which mean the year of 199601 I guessed.

However, I finally took the .values back to 2)., which means using pd.to_datetime(dataset['time'].values, format='%Y%m') , I got the things like ‘1996-01-01T00:00:00.000000000’, which is exactly what I want.
