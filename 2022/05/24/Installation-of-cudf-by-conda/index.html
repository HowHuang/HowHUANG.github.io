When I wanted to install cudf in my old conda environment (this env only had installed necessary package like py-xgboost-gpu), it failed with system conflict error:

The machine was installed cuda-11.7 (I thought I had updated the version of cuda because I met another issue before, but I forgot what is the issue). Based on, the official document recommends

I did three kinds of tries according to searching results:

Remove miniconda thoroughly and install anaconda

(both of them were of a lateset version)

ref. https://github.com/rapidsai/cudf/issues/8414

This try could not fix the conflict.

Update the version of glibc from GNU

  1. The software versions are listed in https://ftp.gnu.org/gnu/glibc/ (I chose 2.35)
  2. Take an out-of-source build

(bashrc for non-loginshell，and profile for login shell)

Updating glibc is highly risky since it is a very central package.

Moreover, by default, the highest version of GLIBC for Ubuntu 18.04 should 2.27. You have two options to continue (ref.)

Restrict the version fo cudf

  1. create a new environment of python=3.7

  2. conda install -c nvidia -c rapidsai -c numba -c conda-forge -c defaults cudf=0.6

Others

In the other situation, I can not use GPU because of outdated libxgboost.so (https://stackoverflow.com/questions/65103861/how-to-install-xgboost-on-macos-compiled-with-gpu-support). In this case, need to compile and install xgboost,
cmake …/xgboost/ -DUSE_CUDA=ON
make -j
cd …/xgboost/python-package/
python setup.py install
(taking an out-of-source build) (by default USE_CUDA=OFF, which means “Build with GPU acceleration” OFF)
During this compiling, I got No CMAKE_CUDA_COMPILER could be found. I just,
export PATH=/usr/local/cuda/bin:$PATH

if necessary, you need to update CUDA version (https://developer.nvidia.com/cuda-downloads?target_os=Linux&target_arch=x86_64&Distribution=Ubuntu&target_version=18.04&target_type=runfile_local).
