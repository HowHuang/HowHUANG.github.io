This is a Centos7 machine

The error raised by:

ifort: /lib64/libstdc++.so.6: version `GLIBCXX_3.4.21’ not found (required by ifort)

There seems two ways to solve it, according to https://stackoverflow.com/questions/44205687/glibcxx-3-4-21-not-found-on-centos-7.

One way is to build a higher version gcc for providing a libstdc++.so.6 and libstdc++.so.6.0.24; The second way is to install gcc by conda and then modify LD_LIBRARY_PATH.

I try the first way by installing gcc-8.3.0, referring to https://www.cnblogs.com/NanZhiHan/p/11010130.html and https://blog.csdn.net/weixin_44575514/article/details/105842062. That is:
