When library(rjags), failing to load the package since

  1. To solve it, firstly need to install jags by homebrew

    running this code by Terminal (see https://macappstore.org/jags/)

    then can find jags in /opt/homebrew/opt/jags

  2. Then install rjags by devtools

    The following code was modified from https://gist.github.com/casallas/8411082

    source code can also be found in https://cran.r-project.org/web/packages/rjags/index.html

  3. Actrually this is because that rjags package is an interface to the JAGS library.

    See https://stackoverflow.com/questions/70533673/error-package-or-namespace-load-failed-for-rjags

    So, for Mac M1 chips, another method is to download JAGS from https://sourceforge.net/p/mcmc-jags/support-requests/8/, and then directly to install rjags by install.packages("rjags") (since this installing method will load the dynamic library from JAGS, not homebrew’s jags)
