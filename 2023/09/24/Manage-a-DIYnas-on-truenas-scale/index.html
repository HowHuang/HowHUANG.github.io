In my nas with turenas-scale 22.12.2, I would like to create a dataset to be mounted on a MacOS/linux machine. Two connection ways could be chosen: NFS and SMB.

I first tried to use NFS since it is marked with UNIX and seems more suitable for mac or linux. In this way, I turned on UNIX (NFS) Shares on the truenas scale system, and then mounted it on MacOS. However, there would be RPC failed:: RPC: Program/version mismatch, which is the same as NFS Issue on TrueNAS-SCALE-22.02.0. To solve this problem, I tried the commands of rpc.statd and midclt call nfs.update '{"allow_nonroot": true}', but after calling these commands, the problem still occured.

Then, I saw an idea that it is recommended to use SMBv3 shares with recent MacOS releases, so I tried to use SMB as the following.

  1. Turn on Windows (SMB) Shares on the truenas scale system. The Purpose could be Default share parameters, and login users (which can be created in Credentials-Local Users) should have permissions for the datasets chosen to be shared.

  2. An easy way for mac to connect the SMB datasets is use folder app.
